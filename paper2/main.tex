\begin{abstract}
Traditional API authorization requires complex infrastructure: OAuth servers, databases for session management, subscription services, and payment processors.
This creates operational overhead, vendor lock-in, and single points of failure.
We present EVMAuth, a complete on-chain authorization framework that eliminates these dependencies by using blockchain token ownership as the authorization primitive.
EVMAuth provides seven composable features: role-based access control, account freezing, token enumeration, time-to-live expiration, transfer control, direct purchase, and dynamic configuration.
We implement EVMAuth in both ERC-1155 and ERC-6909 token standards and deploy to Base testnet.
Our evaluation shows that EVMAuth reduces authorization infrastructure from multiple services to a single smart contract while maintaining sub-100ms verification latency.
Gas costs scale with O(k) for most operations and O(kÂ²) for worst-case transfers, where k is a configurable storage bound.
We demonstrate real-world applicability through an API gateway integration and discuss implications for AI agent authentication.

\end{abstract}
